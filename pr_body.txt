## Summary
Implements the auto-refund engine (BullMQ worker) for handling virtual account overpayments. Ensures strict compliance with Nomba's integration rules by enforcing a mandatory recipient account lookup before executing transfers, and processes all monetary values exclusively in Kobo to prevent floating-point errors.

## Closes
Closes ICE-304

## Scope
- `src/workers/refund.worker.ts` — Implemented the core BullMQ worker. Integrates `NombaClient` for `lookupBank` and `transferToBank`. Handles success by queuing a webhook delivery. Removed illegal kobo-to-naira conversions and replaced inline error classes with the shared `AppError` from `src/lib/errors.js`.
- `src/repositories/refunds.repo.ts` — Created repository to track refund statuses (`COMPLETED`, `FAILED`) and Nomba transfer references.
- `tests/unit/refund.worker.test.ts` — Added full unit test suite covering the happy path (lookup + exact kobo transfer) and failure state propagation.

## Risk
Medium. This feature initiates real outbound transfers via the Nomba Transfer API. This risk is actively mitigated by:
1. Strict `z.number().int().positive()` Zod validation on the job payload.
2. The `lookupBank` pre-verification step failing the job before a transfer can even be attempted if the account is invalid.
