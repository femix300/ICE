## Summary
Implements the auto-refund engine via BullMQ. This worker processes refunds for overpayments by verifying the recipient's bank account with Nomba, executing the transfer, updating the database, and enqueueing a webhook notification to the merchant.

## Scope
- `src/workers/refund.worker.ts`: Worker implementation. Parses payload with `zod`. Uses `AppError` to capture and wrap Nomba API failures gracefully. 
- `tests/unit/refund.worker.test.ts`: Added full `vitest` suite testing account lookup, transfer execution, validation, and error handling.
- Rebased target branch to `dev`.

## Test Plan
- Unit tests verify complete happy path and failure cases (lookup failure, transfer failure, validation failure).

## Risk
Low. Worker safely handles external API failures via explicit try-catch blocks mapped to `AppError`.
