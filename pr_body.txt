## Summary
To protect merchants and the platform from fraudulent inbound transactions, we need a way to flag suspicious activity (such as rapid velocity spikes or duplicate sender floods) before it impacts the system. This PR introduces a lightweight detection engine designed to act as an early warning system, catching bad actors while keeping false positives low so legitimate pipelines are not interrupted.

## Closes
Closes ICE-308

## Scope
- src/services/anomaly.service.ts: Implemented factory pattern for 4 core rules with resilient per-rule try/catch blocks.
- Configured webhook alerting for flagged anomalies.

## Test Plan
- npm run format && npm run lint && npm run typecheck - verified zero errors.
- npm run test - full unit test coverage achieved for rule triggers and schema validation.

## Risk
Low. Detection logic operates purely as an observer (no mutations). Resilient design ensures that a failure in one detection rule does not halt the entire transaction processing pipeline.
