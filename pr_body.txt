1. **Summary:** 
This PR introduces the merchant webhook URL configuration and API key rotation endpoints. It implements `PUT /v1/merchants/:id/webhook-url` to update the webhook URL with HTTPS validation, and `POST /v1/merchants/:id/api-keys/rotate` to invalidate an existing API key and safely generate a new hashed key. Appropriate tier validation ensures only merchants can access their profiles.

Closes ICE-169

- **Architecture Notes:** Leverages existing `generate` and `hash` functions from `src/lib/api-key.ts` and orchestrates them through `merchants.service.ts` to keep the DB layer abstract. Used `AppError` throughout to comply strictly with ICE engineering rules.
- **Target Branch Deviation:** This PR intentionally targets `feat/merchant-registration` rather than `dev`. This is because P04 depends on P03 (Merchant Registration) which is not yet merged. This "stacked PR" approach ensures the diff stays clean and isolated to only P04 code. Once P03 merges, GitHub will automatically re-target this PR to `dev`.
