## Summary
Implements a rule-based anomaly detector service designed to detect suspicious inbound payment patterns. Alerts are queued to the merchant webhook on every triggered rule.

## Scope
- `src/services/anomaly.service.ts`: Implemented 4 base rules: `velocity_spike`, `round_number_flooding`, `duplicate_sender`, and `dormant_account_payment`. Enforced Zod schema parsing on the incoming raw payload and `AppError` on invalid payloads. Implemented resilient loop checking that captures inner rule failures without halting evaluation.
- `tests/unit/anomaly.service.test.ts`: Added full `vitest` suite covering schema validation, single rule triggers, multiple triggers, and loop resilience.
- Rebased target branch to `dev`.

## Test Plan
- Unit tests verify all core rule triggers and error handling logic using mocked dependencies.

## Risk
Low. This is an asynchronous detection service that only raises alerts via webhooks and does not mutate core entities.
